A 2D Adventure Game Engine with Tilemap Editor
Lost Bunny: Journey Home - Game Engine Documentation

Introduction

Lost Bunny: Journey Home is a 2D adventure game engine built in the D programming language using SDL3 for graphics and input handling. The game tells the story of a pet bunny that has been hit by a car and finds itself in the spirit realm. The player must navigate through three mystical levels to find their way back to their body in the real world.

Features

  • Entity-Component-System (ECS) Architecture: Flexible and efficient game object management
  • Scene Management: System for managing different game states (menu, gameplay, cutscenes)
  • Tilemap System: 2D level rendering with JSON-based level files
  • Resource Management: Centralized loading and management of textures and sounds
  • Camera System: World-to-screen coordinate transformations with following behavior
  • Animation System: Sprite-based animation support
  • Physics: Simple collision detection and response
  • Audio: Sound effects playback

Architecture Overview

The engine follows a modular design with the following key components:

  • GameApplication (gameapplication.d): Main application class that manages the game loop, initialization, and scene management
  • Component System (component_system.d): ECS implementation with Transform, Texture, Animation, and Script components
  • Scene System (menuscene.d, lostbunnyscene.d, etc.): Different game states
  • Resource Manager (resourcemanager.d): Loads and manages game assets
  • Tilemap System (tilemap.d, tileset.d): 2D level rendering and collision

Building the Project

Prerequisites:

  • D compiler (DMD or LDC2)
  • DUB (D package manager)
  • SDL3 library

To build the game:

cd Engine
dub build --config=application
./lostbunny

To build the tilemap editor:

cd Engine/Editor
dub build --config=editor
./tilemap_editor

Game Controls

Menu:

  • Arrow Keys / W/S: Navigate menu options
  • Enter / Space: Select menu option

In-Game:

  • Right & Left Arrow Keys / WD: Move the bunny
  • Up Arrow / Space / A: Jump
  • ESC: Return to menu

Tilemap Editor:

  • Q-E: Select level (1-3)
  • 0-8: Select tile type
  • Left Click: Place selected tile
  • Right Click: Erase tile
  • S: Save tilemaps
  • L: Load tilemaps
  • C: Clear entire tilemap

Tile Types

The game uses different tile types for level design:

  • 0: Empty (no tile)
  • 1: Tile (platform block)
  • 2: Spike Up (pointing upward)
  • 3: Spike Right (pointing right)
  • 4: Spike Down (pointing downward)
  • 5: Spike Left (pointing left)
  • 6: Carrot (collectible)
  • 7: Bone (decorative platform tile)
  • 8: Skull (decorative platform tile)

Level Structure

Each level is stored as a JSON file with the following structure:

{
"width": 20,
"height": 15,
"tileSize": 32,
"tiles": [
[0, 0, 0, ...],
[0, 1, 1, ...],
...
]
}

The game world is 640x480 pixels with 32x32 pixel tiles, resulting in a 20x15 tile grid for each level.
